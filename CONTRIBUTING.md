# Contributing to Nodaro

Thanks for your interest in contributing! 🎉

The full contributor guide — dev environment setup, coding standards, the
branching model (`dev` → `main`), commit conventions, testing, and the review
process — lives in **[docs/contributing.md](docs/contributing.md)**.

A few quick pointers:

- **Bugs & features:** open an issue using the templates in the
  [issue chooser](../../issues/new/choose).
- **Security issues:** please do **not** open a public issue — see
  [SECURITY.md](SECURITY.md) for private disclosure.
- **Conduct:** participation is governed by our
  [Code of Conduct](CODE_OF_CONDUCT.md).
- **CLA:** contributions require signing the
  [Contributor License Agreement](CLA.md). The CLA Assistant bot handles this
  automatically on your first pull request.
- **License:** see [LICENSE.md](LICENSE.md) for the three-tier
  (Sustainable Use / Enterprise / Apache-2.0 SDK) overview.

Please read [docs/contributing.md](docs/contributing.md) before opening a pull
request.
